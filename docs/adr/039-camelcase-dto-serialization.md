# ADR-039: camelCase DTO Serialization

**Status:** Accepted
**Date:** 2026-03-16

## Context

Rust uses `snake_case` for fields. JavaScript/TypeScript uses `camelCase`. Without explicit conversion, the frontend gets `undefined` for every field:

```rust
// BAD: no rename_all — serializes as snake_case
#[derive(Serialize)]
pub struct UserResponse {
    pub display_name: String,  // Serializes as "display_name"
    pub avatar_url: Option<String>, // Serializes as "avatar_url"
    pub created_at: String,    // Serializes as "created_at"
}
```

```typescript
// Frontend receives: { "display_name": "Alice", "avatar_url": null, "created_at": "..." }
// But TypeScript generated types expect camelCase:
interface UserResponse {
  displayName: string;  // undefined — field is "display_name" in JSON
  avatarUrl: string | null; // undefined
  createdAt: string; // undefined
}

const user = await api.getMe();
user.displayName; // undefined — silent bug, no runtime error
user.displayName.toUpperCase(); // TypeError in production
```

This bug is insidious because TypeScript says `displayName` is `string` (not `undefined`), so the compiler provides no warning. The bug only surfaces at runtime.

## Decision

All DTOs must include `#[serde(rename_all = "camelCase")]`:

```rust
// GOOD: rename_all = "camelCase" — matches TypeScript conventions
#[derive(Serialize, Deserialize, ToSchema)]
#[serde(rename_all = "camelCase")]
pub struct UserResponse {
    pub display_name: String,  // Serializes as "displayName"
    pub avatar_url: Option<String>, // Serializes as "avatarUrl"
    pub created_at: String,    // Serializes as "createdAt"
}
```

```json
{
  "displayName": "Alice",
  "avatarUrl": null,
  "createdAt": "2026-03-16T12:00:00Z"
}
```

This applies to both request DTOs (`Deserialize`) and response DTOs (`Serialize`). The OpenAPI spec generated by `utoipa` will use camelCase field names, and the auto-generated TypeScript types (ADR-015) will match.

## Consequences

**Positive:**
- Rust `snake_case` and TypeScript `camelCase` conventions are both respected
- Auto-generated TypeScript types match the actual JSON field names
- No silent `undefined` bugs from field name mismatches
- Consistent with JavaScript ecosystem conventions (REST API best practices)

**Negative:**
- Every DTO struct requires the `#[serde(rename_all = "camelCase")]` annotation (enforcement test catches omissions)
- Database column names (`snake_case`) differ from JSON field names (`camelCase`) — this is expected and handled by sqlx at the Rust layer

## Enforcement

- **Enforcement test:** `tests/architecture_test.rs` scans all structs in `src/api/dto/` that derive `Serialize` or `Deserialize` and verifies they include `#[serde(rename_all = "camelCase")]` — test fails if missing
- **OpenAPI verification:** The generated `openapi.json` is checked for camelCase field names in CI
